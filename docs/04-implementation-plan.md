# 実装計画

## 1. 技術方針

サーバー要件が未提示のため、現行環境で動かしやすい PHP 8.2+ と SQLite/MySQL の構成を第一候補とする。公開URLは `public_html` 配下、管理画面も同一アプリ内に置く。依存を増やす前に、ホスティング仕様と本番DBを確認する。

地図は実装を差し替え可能にし、座標データを特定サービス固有形式にしない。初期候補はOpenStreetMap系タイル＋Leafletだが、公開前に利用規約、表示量、帰属表示、運用コストを確認する。

## 2. 実装フェーズ

### Phase 1: 土台

- アプリ構成、環境設定、DBスキーマ、認証
- 共通レイアウト、デザイントークン、レスポンシブナビゲーション
- メディアアップロード基盤

### Phase 2: 運用機能

- HERO管理とライブプレビュー
- 施工事例CRUD、複数画像、座標入力・地図プレビュー
- 最新情報CRUD、予約公開
- サイト基本設定

### Phase 3: 公開サイト

- ホームと3枚HEROスライダー
- サービス、私たちについて、会社案内
- 施工事例一覧・詳細・沖縄マップ
- 最新情報一覧・詳細
- 問い合わせ、個人情報保護方針

### Phase 4: 品質と公開準備

- アクセシビリティ、SEO、画像最適化、性能改善
- セキュリティ試験、フォーム試験、権限試験
- 初期コンテンツ投入、バックアップ／復元確認
- 本番環境設定と公開前チェック

## 3. MVP受け入れ条件

- 管理者が3枚のHERO画像を変更し、順番と公開状態を変えられる
- 各HEROのオーバーレイとドット柄を管理画面で調整し、プレビューできる
- 公開ページでHEROが自動・手動操作でき、キーボード操作にも対応する
- 管理者が施工事例に緯度・経度を登録し、公開側の沖縄マップに反映できる
- 管理者が最新情報を下書き、予約、公開できる
- 公開サイトが主要なスマートフォン・PC幅で破綻しない
- 問い合わせが検証・迷惑送信対策を経て管理者へ届く
- 未確定の会社情報や画像が管理画面から差し替え可能で、コード修正を要しない

## 4. 実装開始前の確認ゲート

- 本番ホスティングのPHP・DB・メール送信仕様
- ロゴと本部ブランドガイドの有無
- 正式な会社・連絡先情報
- 地図サービスの選定

上記が未確定でも、プレースホルダーと設定値でMVP開発は開始できる。公開だけは確定後に行う。
